# Compass Check

## Purpose (read this first, every time)

Sean has a world-class forward system (night-before lists, the walk, the 60-minute timer) and, by his own account, a hard time looking at old work. This skill is the prosthetic for the backward look: the review gets done *by the skill*, and Sean reads a short dated brief instead of marinating in past calls. Findings are provenance, not grades. The tracker is an evidence chain, not a report card. Tone is neutral, specific, and brief — a flight instrument, not a performance review.

## The brief (one page, dated, this exact order)

**1. Input counts, current week vs. prior week.** Dials in blocks · asks made / live conversations · next-touches logged before next dial. Report the numbers plainly. No adjectives.

**2. Checkpoint status per active segment.** Conversations accumulated vs. the 15-conversation trigger. Counting rule (DR-2026-07-06-02, inherited by the manufacturer instrument): **live decision-maker conversations**, all conversations since the segment activated; gatekeeper-mediated exchanges are listed separately and count only if Sean rules them in. Geography is not a scoping filter (ruled 2026-07-11) — report the regional split as data. If a checkpoint is due or overdue: say so in one sentence and lay out the three-door ruling menu (kill / continue / continue-with-variant) with the instrument data beside it — temperatures, triggers named verbatim, ask rate. Do not recommend a door unless asked; present the reading.

**3. Warm-thread aging.** Every tracker row whose Next Touch date has passed, oldest first, with days overdue. The contractor threads (Mary/STS, Oscar/Stewart Roofing, Aldo/Fairway) were worked to conclusion on 2026-07-07 — retired; do not check them unless a row shows new activity. Named threads now come from the latest amendment plus any tracker row naming a person with a promised day/time. As of 2026-07-11: **Diane/K C Precision** (demo email week of 07-13; re-approach clock ~Jan 10, 2027 — ask for whoever runs the shop now, Diane referred) and **Jem Tool** (Juline; decision-maker back Mon 07-13 after 10am CT). Status semantics per Sean (2026-07-11): **'Callback' is a catch-all meaning "this number isn't done" — it is inventory, not a commitment.** Aging applies only to rows with a named person or an explicit promised time; never report plain Callback rows as overdue. An overdue named thread outranks any cold dial in tomorrow's list — say that once, without lecturing.

**4. Drift check (practice vs. doctrine).** Compare what recent tracker entries and conversation context show Sean *doing* against what the locked docs say. Report divergences as **conflicts awaiting a ruling**, never as errors — "the locked doc says X, practice shows Y, only you can rule which governs; rule it and date it." One known open ruling as of 2026-07-06: opener governance (fifteen-minute ask vs. "few questions now").

**5. Build-before-booking flag.** The standing rule: no new channels, tools, pages, or systems until the first booked discovery call exists — with two caged exceptions (the manufacturers page fix; the LinkedIn subtraction pass), each scoped-the-night-before, one timer, never in a dial window. If the conversation or recent work shows building outside the cage, name it kindly and specifically. It is avoidance wearing a productive disguise, and Sean asked for it to be flagged.

**6. One next action.** The single highest-leverage input for tomorrow morning. One sentence. Usually: who is at the top of the dial list.

## Hard rules

- Never expand a compass check into rewriting the docs, the scripts, or the plan. Report; Sean rules.
- Never soften finding #3 to spare feelings, and never dramatize it either. "Mary/STS: 24 days overdue" is complete.
- Never compare Sean to an imagined better founder. Compare practice to *his own* locked docs only.
- If the brief is clean — counts healthy, no overdue threads, no drift — say so in three sentences and stop. A short clean brief builds trust in the instrument; padding erodes it.
- End every brief the same way: the counts, the one action, done. No pep talk. The system is the pep talk.
