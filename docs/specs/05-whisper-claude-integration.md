## Purpose of this document

The post-call processing pipeline: download the recording, transcribe locally with faster-whisper, write the transcript to a designated folder. Coaching feedback is generated by Claude Desktop via filesystem MCP — see spec 09 for that workflow.

## Why this design (no Claude API)

The original Phase 1 design used the Anthropic API for per-call coaching feedback. This was changed before build started because:

- **Cost:** Claude subscription is fixed; API is variable. Even at ~$5/month projected, that matters with runway exhausted.
- **Simplicity:** No API key, no model string verification, no token counting, no cost tracking, no API failure modes.
- **Better coaching experience:** Claude Desktop is interactive — Sean can ask follow-up questions, spot patterns across calls, refine the conversation. API approach is one-shot.
- **Original mental model:** Sean's design intuition from early conversations was "MCP-everything" with Claude Desktop as the second viewpoint. The API approach was a Designer drift that got corrected before code shipped.

## Pipeline overview

```
User clicks "Process Call"
  │
  ▼
1. Download recording from Twilio URL → storage/recordings/{call_id}.mp3
  │
  ▼
2. Run faster-whisper subprocess on mp3 → transcript text
  │
  ▼
3. Save transcript to storage/coaching/transcripts/{call_id}.md
  │  (Also save to calls.transcript column for fast display)
  │
  ▼
4. Set processed_at = now()
  │
  ▼
Redirect user to call detail page
```

That's the pipeline. Everything after step 4 happens in Claude Desktop, not the dashboard.

For Phase 1, this entire pipeline runs synchronously in the request. A 5-minute call processes in ~60-90 seconds. Show a loading spinner. Phase 2 should move this to a queued job.

**Browser timeout note:** Use a regular form POST (not fetch) for Process Call. Browsers wait indefinitely on form submissions; fetch has default timeouts (~60s in some browsers) that can fire before transcription completes.

**Idempotency:** The pipeline is designed so that clicking Process Call twice doesn't redo successful work. Each step checks if its output exists before running. This means failed runs can be retried (the failed step runs, the previous successful steps skip).

## Folder structure

The dashboard manages two coaching-related folders:

```
storage/
└── coaching/
    ├── transcripts/
    │   ├── 1.md
    │   ├── 2.md
    │   └── ...
    └── feedback/
        ├── 1.md  (written by Claude Desktop, optional)
        ├── 3.md  (some calls may not have coaching yet)
        └── ...
```

- **`transcripts/`** — written by the dashboard after Whisper completes. One file per call, named by call_id.
- **`feedback/`** — written by Claude Desktop via filesystem MCP. Same naming convention. Not all transcripts will have feedback files; Claude Desktop processes them on demand.

The dashboard reads from `feedback/` to display coaching on the call detail page. If a feedback file exists for a call, render it. If not, display "Not yet coached — run Claude Desktop on this transcript."

## Component 1: Recording download

`app/Services/RecordingDownloader.php`:

```php
<?php

namespace App\Services;

use App\Models\Call;
use Illuminate\Support\Facades\Http;

class RecordingDownloader
{
    public function download(Call $call): string
    {
        if ($call->recording_local_path && file_exists($call->recording_local_path)) {
            return $call->recording_local_path;
        }

        if (!$call->recording_url) {
            throw new \Exception('No recording URL on call.');
        }

        $accountSid = config('services.twilio.account_sid');
        $authToken = config('services.twilio.auth_token');

        // Twilio recording URLs require basic auth
        $response = Http::withBasicAuth($accountSid, $authToken)
            ->timeout(60)
            ->get($call->recording_url);

        if (!$response->successful()) {
            throw new \Exception('Failed to download recording: ' . $response->status());
        }

        $directory = storage_path('app/recordings');
        if (!is_dir($directory)) {
            mkdir($directory, 0755, true);
        }

        $path = $directory . '/' . $call->id . '.mp3';
        file_put_contents($path, $response->body());

        $call->update(['recording_local_path' => $path]);

        return $path;
    }
}
```

## Component 2: Whisper transcription

`scripts/transcribe.py`:

```python
#!/usr/bin/env python3
"""Transcribe an audio file using faster-whisper. Prints transcript to stdout."""

import sys
import os
from faster_whisper import WhisperModel


def format_timestamp(seconds):
    mins = int(seconds // 60)
    secs = int(seconds % 60)
    return f"{mins:02d}:{secs:02d}"


def main():
    if len(sys.argv) < 2:
        print("Usage: transcribe.py <audio_file_path>", file=sys.stderr)
        sys.exit(1)

    audio_path = sys.argv[1]
    if not os.path.exists(audio_path):
        print(f"File not found: {audio_path}", file=sys.stderr)
        sys.exit(1)

    model_size = os.environ.get("WHISPER_MODEL", "small")
    device = os.environ.get("WHISPER_DEVICE", "cpu")
    compute_type = os.environ.get("WHISPER_COMPUTE_TYPE", "int8")
    # Default: auto-detect language. Override with WHISPER_LANGUAGE env var if needed.
    language = os.environ.get("WHISPER_LANGUAGE")  # None = auto-detect

    # Initialize model. First run downloads from HuggingFace (~460MB for small).
    model = WhisperModel(model_size, device=device, compute_type=compute_type)

    # Transcribe. The recording is dual-channel (left=user, right=lead);
    # faster-whisper mixes channels to mono via ffmpeg by default.
    transcribe_kwargs = {
        "beam_size": 5,
        "vad_filter": True,  # voice activity detection — skip silence
    }
    if language:
        transcribe_kwargs["language"] = language

    segments, info = model.transcribe(audio_path, **transcribe_kwargs)

    # Output: timestamped lines, one per segment
    for segment in segments:
        start = format_timestamp(segment.start)
        end = format_timestamp(segment.end)
        print(f"[{start} - {end}] {segment.text.strip()}")


if __name__ == "__main__":
    main()
```

Make it executable: `chmod +x scripts/transcribe.py`

`app/Services/Transcriber.php`:

```php
<?php

namespace App\Services;

use App\Models\Call;
use App\Models\Lead;
use Symfony\Component\Process\Process;
use Symfony\Component\Process\Exception\ProcessFailedException;

class Transcriber
{
    public function transcribe(Call $call): string
    {
        if (!$call->recording_local_path) {
            throw new \Exception('Recording not downloaded yet.');
        }

        $script = base_path('scripts/transcribe.py');
        $python = config('services.whisper.python_path', 'python3');

        $process = new Process([
            $python,
            $script,
            $call->recording_local_path,
        ]);

        $env = [
            'WHISPER_MODEL' => config('services.whisper.model', 'small'),
            'WHISPER_DEVICE' => config('services.whisper.device', 'cpu'),
            'WHISPER_COMPUTE_TYPE' => config('services.whisper.compute_type', 'int8'),
        ];
        $language = config('services.whisper.language');
        if ($language) {
            $env['WHISPER_LANGUAGE'] = $language;
        }

        $process->setEnv($env);
        $process->setTimeout(300);  // 5 minutes max for transcription
        $process->run();

        if (!$process->isSuccessful()) {
            throw new ProcessFailedException($process);
        }

        $transcript = $process->getOutput();

        if (empty(trim($transcript))) {
            throw new \Exception('Whisper returned empty transcript.');
        }

        // Save to DB (for fast display)
        $call->update(['transcript' => $transcript]);

        // Save to coaching folder (for Claude Desktop to read via MCP)
        $this->writeTranscriptFile($call, $transcript);

        return $transcript;
    }

    private function writeTranscriptFile(Call $call, string $transcript): void
    {
        $directory = storage_path('app/coaching/transcripts');
        if (!is_dir($directory)) {
            mkdir($directory, 0755, true);
        }

        $lead = $call->lead;
        $duration = $call->duration_seconds ?? 0;
        $disposition = $call->disposition ?? 'unknown';
        $painPoints = $call->pain_points ?? '(none captured)';
        $notes = $call->notes ?? '(none)';

        // Markdown with frontmatter — Claude Desktop reads this whole file
        $content = <<<MARKDOWN
---
call_id: {$call->id}
business_name: {$lead->business_name}
contact_name: {$lead->contact_name}
industry: {$lead->industry}
disposition: {$disposition}
duration_seconds: {$duration}
date: {$call->created_at->toIso8601String()}
---

# Call Transcript: {$lead->business_name}

**Disposition:** {$disposition}
**Duration:** {$duration} seconds

## Pain points captured
{$painPoints}

## Sean's own notes
{$notes}

## Transcript
{$transcript}
MARKDOWN;

        file_put_contents($directory . '/' . $call->id . '.md', $content);
    }
}
```

## Component 4: The orchestrator

`app/Http/Controllers/CallController.php` — `process()` method:

```php
public function process(Call $call, RecordingDownloader $downloader, Transcriber $transcriber)
{
    try {
        if (!$call->recording_local_path) {
            $downloader->download($call);
            $call->refresh();
        }

        if (!$call->transcript) {
            $transcriber->transcribe($call);
            $call->refresh();
        }

        if (!$call->processed_at) {
            $call->update(['processed_at' => now()]);
        }

        return redirect()->route('calls.show', $call)
            ->with('success', 'Call transcribed. Open Claude Desktop to coach.');

    } catch (\Exception $e) {
        \Log::error('Process call failed', ['call_id' => $call->id, 'error' => $e->getMessage()]);
        return redirect()->route('calls.show', $call)
            ->with('error', 'Processing failed: ' . $e->getMessage());
    }
}
```

Note: no `CoachingGenerator` injection. The pipeline ends at transcription.

## What the user sees

When user clicks "Process Call":
1. Form POST to `/calls/{id}/process`
2. Server takes 60-90 seconds (download ~5s, transcribe ~30-60s)
3. Browser shows loading state
4. Server redirects to `/calls/{id}` with success flash
5. User sees full transcript on call detail page
6. Coaching section says "Not yet coached" with instructions

User then opens Claude Desktop separately to generate coaching. See spec 09.

## Performance expectations

Rough benchmarks for OptiPlex 9020 MT class hardware (4-core Haswell CPU, no GPU, 32GB RAM):

| Audio length | faster-whisper small | Total pipeline |
|---|---|---|
| 1 minute | ~10-15 seconds | ~25-30 seconds |
| 5 minutes | ~45-60 seconds | ~75-90 seconds |
| 15 minutes | ~2-3 minutes | ~3-4 minutes |

Memory: faster-whisper uses ~1.5GB peak with int8 quantization on `small`.

## Failure modes and recovery

- **Recording not yet uploaded to Twilio:** the recording webhook may take 30-60 seconds after hangup, sometimes longer. UI should hide the Process button until the recording webhook fires.
- **Network hiccup during download:** retry once. If still failing, leave recording_local_path null and show error. User can click Process Call again — orchestrator's idempotency means previous successful steps don't redo.
- **Whisper subprocess crashes:** rare but possible. Show error, leave transcript null. User can retry.
- **Empty transcript:** if the recording is silent or Whisper produces nothing, treat as a failure. Surface error.
- **Coaching folder doesn't exist:** auto-created when first transcript is written. Same with feedback folder when Claude Desktop tries to write.

## Cost expectations

faster-whisper local: $0
Coaching via Claude Desktop subscription: $0 incremental (uses existing subscription)

Total Phase 1 coaching cost: $0/month. This was the main reason for the architecture change.

## Out of scope for Phase 1

- Speaker diarization — faster-whisper outputs flat text. Phase 2 could process channels separately.
- Real-time / streaming transcription — Phase 1 is post-call only.
- Queue-based async processing — synchronous is fine for one user processing calls in sequence.
- Auto-generated coaching from API — explicitly removed in favor of Claude Desktop workflow.
- Multiple frameworks per call — Phase 2 (would mean multiple feedback files per call).
