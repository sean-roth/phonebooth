## Purpose of this document

The post-call processing pipeline: download the recording, transcribe each channel separately with faster-whisper to preserve speaker attribution, write the transcript to a designated folder. Coaching feedback is generated by Claude Desktop via filesystem MCP — see spec 09.

## Why dual-channel matters (don't mix to mono)

Twilio's `record-from-answer-dual` mode produces a stereo file: left channel = Sean (the cockpit user), right channel = the lead. This is the entire point of dual-channel recording.

The naive approach is to let faster-whisper mix the channels to mono before transcribing. This produces a flat transcript like:

```
[00:14] Hi, this is Sean Roth from...
[00:18] Yeah, what's this about?
[00:21] I'm calling because I noticed...
```

Without speaker attribution, **coaching feedback can only describe the call abstractly**. It can't say "you opened weakly here" or "he gave you a buying signal at 1:43 and you missed it" because there's no way to know who said what. The whole point of the system — coaching feedback against a sales framework — degrades to generic observations.

The fix: transcribe each channel separately, then merge timestamps to produce an attributed transcript:

```
[00:14] SEAN: Hi, this is Sean Roth from...
[00:18] LEAD: Yeah, what's this about?
[00:21] SEAN: I'm calling because I noticed...
```

This is harder than mono transcription (two Whisper passes, then merge), but the coaching quality difference is dramatic.

## Why this design (no Claude API)

Coaching feedback is generated by Claude Desktop via filesystem MCP, not the dashboard via API. See spec 09. This was an architectural correction made before build started:

- **Cost:** Claude subscription is fixed; API is variable
- **Simplicity:** No API key, no model string, no token counting, no cost tracking
- **Better coaching:** Claude Desktop is interactive — Sean can ask follow-ups, spot patterns across calls

## Pipeline overview

```
User clicks "Process Call"
  │
  ▼
1. Download recording from Twilio URL → storage/recordings/{call_id}.mp3
  │
  ▼
2. Split stereo recording into two mono files via ffmpeg:
     storage/recordings/{call_id}_left.mp3 (Sean)
     storage/recordings/{call_id}_right.mp3 (lead)
  │
  ▼
3. Run faster-whisper on each channel
  │
  ▼
4. Merge segments by timestamp → attributed transcript
  │
  ▼
5. Save transcript to calls.transcript column AND
     storage/coaching/transcripts/{call_id}.md
  │
  ▼
6. Set processed_at = now()
```

Approximate timing: 5-minute call takes ~90-120 seconds total (slightly longer than mono because of two Whisper passes, but each pass works on half the audio length effectively, so the slowdown is more like 1.3x not 2x).

For Phase 1, the pipeline runs synchronously. Form POST (not fetch) for browser timeout reasons.

**Don't process declined-recording calls.** If `disposition === 'declined_recording'`, the Process Call action is rejected (the recording has been deleted). See spec 03.

## Component 1: Recording download

Same as before — download the stereo MP3 from Twilio with basic auth. See `RecordingDownloader.php` below.

```php
<?php

namespace App\Services;

use App\Models\Call;
use Illuminate\Support\Facades\Http;

class RecordingDownloader
{
    public function download(Call $call): string
    {
        if ($call->recording_local_path && file_exists($call->recording_local_path)) {
            return $call->recording_local_path;
        }

        if (!$call->recording_url) {
            throw new \Exception('No recording URL on call.');
        }

        $accountSid = config('services.twilio.account_sid');
        $authToken = config('services.twilio.auth_token');

        $response = Http::withBasicAuth($accountSid, $authToken)
            ->timeout(60)
            ->get($call->recording_url);

        if (!$response->successful()) {
            throw new \Exception('Failed to download recording: ' . $response->status());
        }

        $directory = storage_path('app/recordings');
        if (!is_dir($directory)) {
            mkdir($directory, 0755, true);
        }

        $path = $directory . '/' . $call->id . '.mp3';
        file_put_contents($path, $response->body());

        $call->update(['recording_local_path' => $path]);

        return $path;
    }
}
```

## Component 2: Channel splitter

Splits the stereo MP3 into two mono files using ffmpeg. ffmpeg is already required by faster-whisper, so no new dependency.

`app/Services/ChannelSplitter.php`:

```php
<?php

namespace App\Services;

use App\Models\Call;
use Symfony\Component\Process\Process;
use Symfony\Component\Process\Exception\ProcessFailedException;

class ChannelSplitter
{
    /**
     * Split a stereo recording into two mono files.
     *
     * @return array{left: string, right: string} Paths to the split files
     */
    public function split(Call $call): array
    {
        $sourcePath = $call->recording_local_path;
        if (!$sourcePath || !file_exists($sourcePath)) {
            throw new \Exception('Source recording not found.');
        }

        $directory = dirname($sourcePath);
        $base = $call->id;

        $leftPath = "{$directory}/{$base}_left.mp3";
        $rightPath = "{$directory}/{$base}_right.mp3";

        if (file_exists($leftPath) && file_exists($rightPath)) {
            return ['left' => $leftPath, 'right' => $rightPath];
        }

        // ffmpeg command: extract left channel (pan=mono|c0=c0) and right (pan=mono|c0=c1)
        // -y overwrites existing files, -loglevel error keeps stdout quiet
        $process = new Process([
            'ffmpeg',
            '-y',
            '-loglevel', 'error',
            '-i', $sourcePath,
            '-map_channel', '0.0.0',
            $leftPath,
            '-map_channel', '0.0.1',
            $rightPath,
        ]);

        $process->setTimeout(60);
        $process->run();

        if (!$process->isSuccessful()) {
            throw new ProcessFailedException($process);
        }

        return ['left' => $leftPath, 'right' => $rightPath];
    }
}
```

**Verification note:** the `ffmpeg -map_channel` syntax is a known-working pattern but ffmpeg has multiple ways to do channel splitting. The Engineer should verify this command produces clean mono outputs (test with `ffprobe storage/app/recordings/{id}_left.mp3` — should report 1 audio channel).

## Component 3: Whisper transcription script

Updated to process a single mono file and emit JSON segments (so the merger can read them programmatically).

`scripts/transcribe.py`:

```python
#!/usr/bin/env python3
"""
Transcribe a mono audio file using faster-whisper.
Outputs JSON to stdout: a list of {start, end, text} segments.
"""

import sys
import os
import json
from faster_whisper import WhisperModel


def main():
    if len(sys.argv) < 2:
        print("Usage: transcribe.py <audio_file_path>", file=sys.stderr)
        sys.exit(1)

    audio_path = sys.argv[1]
    if not os.path.exists(audio_path):
        print(f"File not found: {audio_path}", file=sys.stderr)
        sys.exit(1)

    model_size = os.environ.get("WHISPER_MODEL", "small")
    device = os.environ.get("WHISPER_DEVICE", "cpu")
    compute_type = os.environ.get("WHISPER_COMPUTE_TYPE", "int8")
    language = os.environ.get("WHISPER_LANGUAGE")  # None = auto-detect

    model = WhisperModel(model_size, device=device, compute_type=compute_type)

    transcribe_kwargs = {
        "beam_size": 5,
        "vad_filter": True,
    }
    if language:
        transcribe_kwargs["language"] = language

    segments, info = model.transcribe(audio_path, **transcribe_kwargs)

    # Output as JSON array — merger reads this
    output = [
        {
            "start": round(segment.start, 2),
            "end": round(segment.end, 2),
            "text": segment.text.strip(),
        }
        for segment in segments
    ]

    print(json.dumps(output))


if __name__ == "__main__":
    main()
```

`chmod +x scripts/transcribe.py`

## Component 4: Transcriber (orchestrates split + dual transcription + merge)

`app/Services/Transcriber.php`:

```php
<?php

namespace App\Services;

use App\Models\Call;
use Symfony\Component\Process\Process;
use Symfony\Component\Process\Exception\ProcessFailedException;

class Transcriber
{
    public function __construct(
        private ChannelSplitter $splitter,
    ) {}

    public function transcribe(Call $call): string
    {
        if (!$call->recording_local_path) {
            throw new \Exception('Recording not downloaded yet.');
        }

        // Step 1: split stereo into two mono files
        $channels = $this->splitter->split($call);

        // Step 2: transcribe each channel separately
        $leftSegments = $this->transcribeChannel($channels['left']);
        $rightSegments = $this->transcribeChannel($channels['right']);

        // Step 3: merge by timestamp, with speaker labels
        $transcript = $this->mergeSegments($leftSegments, $rightSegments);

        if (empty(trim($transcript))) {
            throw new \Exception('Whisper returned empty transcript on both channels.');
        }

        // Step 4: save to DB and to coaching folder
        $call->update(['transcript' => $transcript]);
        $this->writeTranscriptFile($call, $transcript);

        return $transcript;
    }

    private function transcribeChannel(string $audioPath): array
    {
        $script = base_path('scripts/transcribe.py');
        $python = config('services.whisper.python_path', 'python3');

        $process = new Process([$python, $script, $audioPath]);

        $env = [
            'WHISPER_MODEL' => config('services.whisper.model', 'small'),
            'WHISPER_DEVICE' => config('services.whisper.device', 'cpu'),
            'WHISPER_COMPUTE_TYPE' => config('services.whisper.compute_type', 'int8'),
        ];
        $language = config('services.whisper.language');
        if ($language) {
            $env['WHISPER_LANGUAGE'] = $language;
        }

        $process->setEnv($env);
        $process->setTimeout(300);
        $process->run();

        if (!$process->isSuccessful()) {
            throw new ProcessFailedException($process);
        }

        $output = $process->getOutput();
        $segments = json_decode($output, true);

        if (!is_array($segments)) {
            throw new \Exception('Whisper returned malformed output: ' . substr($output, 0, 200));
        }

        return $segments;
    }

    /**
     * Merge two channel transcripts into a single attributed transcript.
     * Left channel = Sean, right channel = lead.
     */
    private function mergeSegments(array $leftSegments, array $rightSegments): string
    {
        // Tag each segment with its speaker
        $tagged = [];
        foreach ($leftSegments as $seg) {
            $seg['speaker'] = 'SEAN';
            $tagged[] = $seg;
        }
        foreach ($rightSegments as $seg) {
            $seg['speaker'] = 'LEAD';
            $tagged[] = $seg;
        }

        // Sort by start time
        usort($tagged, fn($a, $b) => $a['start'] <=> $b['start']);

        // Format as timestamped lines
        $lines = [];
        foreach ($tagged as $seg) {
            $start = $this->formatTimestamp($seg['start']);
            $end = $this->formatTimestamp($seg['end']);
            $lines[] = "[{$start} - {$end}] {$seg['speaker']}: {$seg['text']}";
        }

        return implode("\n", $lines);
    }

    private function formatTimestamp(float $seconds): string
    {
        $mins = (int) floor($seconds / 60);
        $secs = (int) ($seconds % 60);
        return sprintf('%02d:%02d', $mins, $secs);
    }

    private function writeTranscriptFile(Call $call, string $transcript): void
    {
        $directory = storage_path('app/coaching/transcripts');
        if (!is_dir($directory)) {
            mkdir($directory, 0755, true);
        }

        $lead = $call->lead;
        $duration = $call->duration_seconds ?? 0;
        $disposition = $call->disposition ?? 'unknown';
        $painPoints = $call->pain_points ?? '(none captured)';
        $notes = $call->notes ?? '(none)';

        $content = <<<MARKDOWN
---
call_id: {$call->id}
business_name: {$lead->business_name}
contact_name: {$lead->contact_name}
industry: {$lead->industry}
disposition: {$disposition}
duration_seconds: {$duration}
date: {$call->created_at->toIso8601String()}
---

# Call Transcript: {$lead->business_name}

**Disposition:** {$disposition}
**Duration:** {$duration} seconds

## Pain points captured
{$painPoints}

## Sean's own notes
{$notes}

## Transcript

Speakers: SEAN (the caller, Sean Roth) and LEAD (the business owner being called).

{$transcript}
MARKDOWN;

        file_put_contents($directory . '/' . $call->id . '.md', $content);
    }
}
```

## Edge case: silent or near-silent channels

If one party barely speaks (e.g., a one-sided rant by the lead, or a voicemail Sean left), one of the two transcription passes may return an empty list. The merge handles this naturally — empty list contributes no segments. Don't fail the whole pipeline if one channel is empty; only fail if **both** are empty.

The check in `transcribe()` does exactly this: it concatenates and trims, and only throws if the combined result is empty.

## Performance expectations

OptiPlex 9020 MT class hardware with two Whisper passes:

| Audio length | Each pass (small, int8) | Total pipeline |
|---|---|---|
| 1 minute | ~10-15 seconds × 2 = 20-30s | ~40-50 seconds |
| 5 minutes | ~45-60 seconds × 2 = 90-120s | ~120-150 seconds |
| 15 minutes | ~2-3 minutes × 2 = 4-6 minutes | ~5-7 minutes |

Memory: each Whisper pass uses ~1.5GB; running serially (not parallel), peak memory is single-pass.

If transcription becomes a bottleneck, Phase 2 options:

1. Run the two passes in parallel (uses ~3GB peak — fine on 32GB OptiPlex)
2. Use the smaller `base.en` model on Sean's channel (he speaks consistently, can use a faster model) and keep `small` for the lead's channel
3. Move to a queued background job

For Phase 1, 2-3 minutes per call processing is acceptable.

## Failure modes and recovery

- **Recording not yet uploaded to Twilio:** UI hides Process button until webhook fires
- **Network hiccup during download:** retry once, leave path null on failure, user retries
- **ffmpeg subprocess crashes:** rare; usually a malformed input file. Fail with clear error.
- **Whisper subprocess crashes:** rare. Fail with clear error.
- **Empty transcript on both channels:** very rare (silent recording); fail with message
- **Empty transcript on one channel only:** continue (other channel has content)
- **Coaching folder doesn't exist:** auto-created when first transcript is written
- **Declined-recording call:** Process Call action rejected upstream in CallController per spec 03

## Out of scope for Phase 1

- Speaker diarization beyond simple channel splitting (e.g., handling crosstalk where both channels have Sean's voice bleeding through)
- Real-time / streaming transcription
- Queue-based async processing
- Auto-coaching from API (explicitly removed)
- Multiple frameworks per call
- Caching: re-running Process Call on the same audio always re-splits and re-transcribes (per the orchestrator's idempotency, this is skipped if outputs exist; but if forced re-run is added in Phase 2, caching the splits saves time)
